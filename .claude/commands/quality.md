# /quality — 코드 품질 검사 명령어

커밋/PR 전에 전체 코드 품질을 점검합니다.

## 검사 항목

1. **컴파일** — `./gradlew compileJava`
2. **빌드** — `./gradlew build` (컴파일 + 테스트 + 모든 검증 포함)
3. **테스트** — `./gradlew test`

> `./gradlew build` 하나면 컴파일·테스트·체크가 모두 수행됩니다. 빠른 피드백이 필요할 때만 위의 세분화된 명령을 사용하세요.

## 결과 보고

```
품질 검사 결과:
[x] 컴파일: PASS
[x] 빌드: PASS
[x] 테스트: PASS (N/M 통과)
```

## 실패 시 행동

- **컴파일 실패**: 원인 파일/라인 식별 후 수정 제안
- **테스트 실패**:
  1. 실패한 테스트와 에러 메시지 확인
  2. 관련 프로덕션 코드 검토
  3. 테스트가 요구하는 동작과 실제 구현의 차이 분석
  4. 수정안 제시 후 사용자 승인 대기

## 커밋 전 필수 실행

이 명령어는 커밋 전에 실행하여 CI가 통과할지 미리 확인합니다. CI(`./gradlew build`)와 동일한 기준입니다.

## 자주 쓰는 옵션

```bash
./gradlew build --no-daemon      # 데몬 이슈 우회
./gradlew test --info            # 테스트 상세 로그
./gradlew clean build            # 깨끗한 상태에서 검증
```
